AI Agent study plan — 12-week curriculum for advanced engineers

12-week curriculum · Advanced track · Build-from-scratch focus
Phase 1 — Agent foundations & mental models
Get precise about what makes something an agent: perception → reasoning → action loops. Bridge from your ML intuition to agentic architectures.
Week 1 · What is an AI agent?
Concepts
Agent taxonomy & the ReAct loop
Reflex vs. goal-based vs. utility-based agents · LLM-as-agent vs. LLM-in-the-loop · The Perceive → Think → Act cycle · Why RAG ≠ agent
📄 Wooldridge & Jennings (1995) 📄 ReAct paper (Yao 2023) 📺 Andrej Karpathy — Software 2.0
Lab
Trace an existing agent end-to-end
Run AutoGPT or LangChain agent · Log every prompt/response · Identify where reasoning happens vs. where tools fire · Annotate the trace manually
Week 2 · LLMs as reasoning engines
Concepts
Prompting for reliable reasoning
Chain-of-thought vs. zero-shot-CoT · Self-consistency decoding · Structured output (JSON mode, tool-use API) · Token budget and context window management
📄 Chain-of-Thought Prompting (Wei 2022) 📄 Anthropic prompt engineering docs
Lab
Build a raw reasoning harness
Call Claude / GPT-4o via API with no framework · Implement CoT yourself · Benchmark structured output reliability · Measure accuracy vs. prompt design
Week 3 · Memory & state
Concepts
Memory taxonomy: in-context, external, episodic
Sliding window context · Vector DB retrieval (FAISS, Chroma, Pinecone) · Semantic vs. episodic memory · Entity extraction & knowledge graphs
📄 MemGPT paper 🛠 LlamaIndex docs
Lab
Persistent memory store from scratch
Store conversation summaries in a vector DB · Build retrieval-augmented recall · Evaluate retrieval precision with a test suite
Milestone: Simple ReAct agent with memory — no framework
Phase 2 — Core agent architecture
Design and build a production-quality single agent: planning, tool use, error handling, and evaluation. This is the heart of the curriculum.
Week 4 · Tool use & function calling
Concepts
Tool design principles
Tool schema design (names matter!) · Parallel vs. sequential tool calls · Tool error propagation · Security: sandboxing code execution, rate limits
📄 Toolformer paper 📄 Claude tool-use API docs
Lab
Build a general-purpose tool registry
Python decorator-based tool registration · Auto-generate JSON schemas from type hints · Implement a retry-with-reflection loop on tool failure
Week 5 · Planning & task decomposition
Concepts
Planning paradigms: MRKL, PAL, tree of thought
Task graph decomposition · Plan-and-execute vs. online planning · Backtracking and replanning · When to replan vs. continue · LLM-as-planner failure modes
📄 Tree of Thoughts (Yao 2023) 📄 Plan-and-Solve (Wang 2023)
Lab
Implement a task graph executor
Parse LLM-generated DAG into a dependency graph · Execute nodes in topological order · Handle partial failures gracefully · Add human-in-the-loop checkpoint
Week 6 · Evaluation & reliability
Concepts
Agentic evals: harder than model evals
Trajectory evaluation vs. outcome evaluation · LLM-as-judge frameworks · Unit testing tools and planners · Fuzzing prompts · Regression suites for agent behavior
📄 AgentBench paper 🛠 Weave / LangSmith / Braintrust
Lab
Build an eval harness for your agent
Define 50 test cases spanning success / failure / edge cases · Auto-grade with LLM judge + manual spot-check · Track metric drift across iterations
Milestone: Evaluated, tested, single-agent system capable of multi-step tasks
Phase 3 — Multi-agent systems
Go beyond a single agent: orchestration, inter-agent communication, role specialization, and emergent coordination patterns.
Week 7 · Multi-agent architecture patterns
Concepts
Orchestrator / subagent / peer topologies
Hub-and-spoke vs. peer mesh · Role specialization (planner, executor, critic, memory) · Shared vs. private context · Trust boundaries between agents · Debate / self-critique loops
📄 Anthropic multi-agent blog 📄 Society of Mind (Minsky)
Lab
Orchestrator + 3 specialist agents
Planner decomposes goals · Researcher fetches/summarizes data · Coder writes and runs code · Critic reviews outputs · All agents share a message bus
Week 8 · Communication & shared state
Concepts
Inter-agent protocols & blackboard architectures
Message passing vs. shared memory · Structured handoff formats (JSON task specs) · Conflict resolution when agents disagree · Async execution & event queues
📄 AutoGen paper (Microsoft 2023) 🛠 LangGraph
Lab
Build a shared blackboard with async agents
Redis-backed shared state · Agents subscribe to task events · Implement locking for concurrent writes · Add a supervisor agent for deadlock detection
Week 9 · Safety, alignment & control
Concepts
Minimal-footprint design & human oversight
Prompt injection attacks · Scope creep / goal misgeneralization · Confirmation checkpoints for irreversible actions · Constitutional constraints in system prompts · Monitoring & kill switches
📄 Anthropic responsible scaling policy 📄 OWASP LLM Top 10
Lab
Red-team your own agent
Inject adversarial tool outputs · Attempt prompt injection via user data · Document attack vectors · Add a guard-rail layer and retest
Milestone: Secure, multi-agent system with observable communication and human oversight
Phase 4 — Production & beyond
Ship it. Observability, latency optimization, cost management, and staying current with fast-moving research.
Week 10 · Observability & debugging
Concepts
Distributed tracing for agent systems
Span-based traces across agent hops · LLM token logging & cost dashboards · Replay debugging (deterministic seeding) · Alerting on reasoning drift
🛠 LangSmith / Braintrust 🛠 OpenTelemetry + Jaeger 🛠 Helicone
Lab
Instrument your agent with full tracing
Add OTEL spans around every LLM call and tool call · Build a cost-per-task dashboard · Set up anomaly alerts · Replay a failed run and fix it
Week 11 · Latency, cost & model selection
Concepts
Optimizing the agent execution budget
Cascade models: small model for routing, large for reasoning · Caching LLM responses (semantic cache) · Prompt compression techniques · Streaming vs. batch execution · Cost/quality Pareto frontier
📄 LLMLingua (prompt compression) 🛠 GPTCache
Lab
Cut cost by 50% without quality loss
Profile token usage per step · Route simple sub-tasks to Haiku/GPT-3.5 · Add semantic caching layer · Measure quality on eval suite before and after
Week 12 · Capstone & staying current
Project
Capstone: end-to-end agent of your choosing
Pick a real domain (coding assistant, research agent, data analyst) · Build from scratch using only your own abstractions · Full eval suite · Observability · Write a postmortem
Meta
Staying current in a fast-moving field
Papers to track: arXiv cs.AI + cs.CL weekly digests · Follow: Lilian Weng's blog, Sebastian Raschka's newsletter · Communities: r/MachineLearning, Hugging Face Discord · Benchmark leaderboards: SWE-bench, GAIA, AgentBench
Milestone: Production-ready capstone agent — shipped, observed, and documented
