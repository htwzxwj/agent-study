Python runtime
Install uv recommended
curl -LsSf https://astral.sh/uv/install.sh | sh source ~/.bashrc
uv replaces pip + venv + pyenv in one tool — 10–100× faster installs. You'll use it to manage Python versions and project dependencies throughout the study plan.
Install Python 3.11 via uv
uv python install 3.11 uv python pin 3.11
3.11 hits the sweet spot: all current agent libraries support it, and it's noticeably faster than 3.10.
Install VS Code WSL extension (on Windows side)
# from your WSL terminal — opens VS Code connected to WSL code .
Install the "WSL" extension in VS Code on Windows first. After that, code . from any WSL directory opens a full Linux-connected VS Code session.
Set up API keys in ~/.bashrc
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.bashrc echo 'export OPENAI_API_KEY="sk-..."' >> ~/.bashrc # optional source ~/.bashrc
Global env vars for your shell means every project inherits them without committing secrets. Each project also gets a .env file loaded via python-dotenv for fine-grained overrides.
Per-project virtual environment
Create your first project with uv
mkdir ~/projects/agent-study && cd ~/projects/agent-study uv init uv venv && source .venv/bin/activate
Install LLM clients Phase 1–4
uv add anthropic openai tiktoken
You'll use the Anthropic SDK as primary. OpenAI is there for comparison experiments in Phase 2.
Install vector DB + memory Phase 1
uv add chromadb faiss-cpu sentence-transformers
ChromaDB for the Week 3 memory store (easy to run locally). FAISS for when you need speed at scale. sentence-transformers for embeddings without an API call.
Install data / async / web stack Phase 2–3
uv add pydantic fastapi uvicorn httpx aiohttp redis python-dotenv
FastAPI for serving agent endpoints. Redis client for the Phase 3 shared blackboard. httpx/aiohttp for async tool calls inside agents.
Install dev / eval tools Phase 2
uv add pytest pytest-asyncio rich loguru jupyter ipykernel
Pytest for your eval harness. Rich for pretty agent traces in the terminal. Loguru for structured logging. Jupyter for exploratory experimentation.
Install observability Phase 4
uv add opentelemetry-api opentelemetry-sdk opentelemetry-exporter-jaeger
Add this when you hit Week 10. Don't install now — you won't need it for Phases 1–3.
Background services via Docker
Install Docker Desktop (Windows) + enable WSL integration
Download Docker Desktop → Settings → Resources → WSL Integration → enable your Ubuntu distro. After this, docker works inside WSL natively.
Run Redis (Phase 3 shared blackboard)
docker run -d --name redis-agent -p 6379:6379 redis:7-alpine
Start this when you reach Phase 3, Week 8. Redis is your shared state store for multi-agent communication.
Run Jaeger (Phase 4 distributed tracing)
docker run -d --name jaeger \ -p 16686:16686 -p 4317:4317 \ jaegertracing/all-in-one:latest
Jaeger UI at localhost:16686. You'll trace every LLM call and tool call through this in Week 10.
Optional: tune WSL2 memory in .wslconfig if RAM < 16GB
# C:\Users\YourName\.wslconfig [wsl2] memory=8GB processors=4 swap=2GB
sentence-transformers loads a ~400MB model into memory. If your machine has 8GB RAM, cap WSL at 6GB to avoid OOM kills.
Claude Code in WSL
Install Node.js 18+ via nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash source ~/.bashrc nvm install 20 && nvm use 20
Configure npm global dir (avoids sudo)
mkdir -p ~/.npm-global npm config set prefix ~/.npm-global echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc source ~/.bashrc
Install Claude Code do NOT use sudo
npm install -g @anthropic-ai/claude-code claude --version
Authenticate and run health check
claude # triggers OAuth login on first run claude doctor # verifies everything is connected
Claude Code is your pair programmer during the lab weeks. Use it inside your project directory — it reads your codebase for context. Especially useful for the eval harness in Week 6 and the multi-agent wiring in Phase 3.
Recommended project structure
Scaffold the study repo
mkdir -p ~/projects/agent-study/{phase1,phase2,phase3,phase4} mkdir -p ~/projects/agent-study/shared/{memory,tools,utils} touch ~/projects/agent-study/.env touch ~/projects/agent-study/shared/__init__.py
Create a shared config loader
# shared/utils/config.py from dotenv import load_dotenv import os load_dotenv() ANTHROPIC_API_KEY = os.environ["ANTHROPIC_API_KEY"] DEFAULT_MODEL = os.getenv("MODEL", "claude-sonnet-4-6") MAX_TOKENS = int(os.getenv("MAX_TOKENS", "4096"))
All phases import from shared/ so you're building on real code, not throwing it away at each phase.
Add a .gitignore
cat > ~/projects/agent-study/.gitignore << 'EOF' .env .venv/ __pycache__/ *.pyc chroma_db/ *.egg-info/ .pytest_cache/ EOF
Verify everything works end-to-end
If you see env ok — your Python env, API key, and SDK are all wired up. You're ready for Week 1.
